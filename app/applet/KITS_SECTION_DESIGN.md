## 3. Recommended Card Layout
- **Visuals:** High-contrast product silhouette or technical line-drawing icon at the top.
- **Tags:** Use monospace, uppercase tags above the title to establish constraints immediately (e.g., `[ 10-YEAR BATTERY ]`, `[ CELLULAR ]`, `[ CLAMP-ON ]`).
- **Structure:** 
  1. Product Name (Large, bold, `Zilla Slab`).
  2. Label/Subtype (Smaller, `stone-500`).
  3. Description (2 lines max).
  4. The "Why this helps" sentence in a slightly muted background box.
  5. Bold CTA button anchored to the bottom.
- **Borders:** Heavy `stone-800` borders with bottom-right drop shadows to make the cards feel like physical equipment boxes.

## 4. Comparison Table Columns (For broader overview)
If displaying a matrix to help users choose between kits:
- **Kit Name**
- **Primary Use** (e.g., Water, Voltage, Access)
- **Install Time** (e.g., "15 minutes", "Under an hour")
- **Power Source** (e.g., "Solar + Lithium", "10-Year Coin Cell")
- **Connectivity** (e.g., "Cellular", "LoRaWAN", "Offline Logger")

## 5. Notes for Kit Detail Pages
- **"What's in the box":** List exactly what ships. No surprises. (e.g., "1x Sensor unit, 2x Heavy-duty zip ties, 1x Activation magnet").
- **"How to install":** A 3-step, idiot-proof list. (1. Clamp to pipe. 2. Pull battery tab. 3. Text 'START' to the number on the box).
- **Specs Without Fluff:** IP67 waterproof rating? Say "Survives submersion and heavy rain." Don't just list IP67.
- **Bridge Back to Math:** Include a link back to the relevant calculator: *"Not sure if it's worth it? Run the numbers on your water loss."*
